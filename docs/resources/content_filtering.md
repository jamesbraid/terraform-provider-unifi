# Content Filtering (Resource)

Manages a UniFi content-filtering policy (UniFi Network 9.x+, v2 API). Content filtering blocks web content by category and by explicit domain lists, optionally enforcing safe search, scoped to specific clients and/or networks. Shown under Settings → Security → Content Filtering in the UniFi UI.

## Example Usage

```terraform
# Block ad and adult content for the kids network, with safe search enforced.
resource "unifi_content_filtering" "kids" {
  name        = "Kids"
  categories  = ["ADVERTISEMENT"]
  network_ids = [unifi_network.kids.id]
  block_list  = ["example-tracker.com"]
  allow_list  = ["example-school.org"]
  safe_search = ["GOOGLE", "YOUTUBE", "BING"]
}
```

## Schema

### Required

- `name` (String) The name of the content-filtering policy.

### Optional

- `allow_list` (Set of String) Domains always allowed, overriding category blocks.
- `block_list` (Set of String) Domains always blocked.
- `categories` (Set of String) Content category tokens to block (e.g. `FAMILY`, `ADVERTISEMENT`). Category tokens come from the controller's content-filtering category list.
- `client_macs` (Set of String) MAC addresses of the clients the policy applies to. May be empty.
- `enabled` (Boolean) Whether the policy is enabled. Defaults to `true`.
- `network_ids` (Set of String) IDs of the networks the policy applies to. May be empty.
- `safe_search` (Set of String) Search providers to enforce safe search for. Observed values: `GOOGLE`, `YOUTUBE`, `BING`.
- `schedule` (Attributes) When the policy is enforced. Only `mode` is modeled; `ALWAYS` is the only mode observed on live controllers so far. (see [below for nested schema](#nestedatt--schedule))
- `site` (String) The name of the UniFi site. Defaults to the site configured in the provider.
- `timeouts` (Attributes) (see [below for nested schema](#nestedatt--timeouts))

The [` + "`" + `terraform import` + "`" + ` command](https://developer.hashicorp.com/terraform/cli/commands/import) can be used, for example:
